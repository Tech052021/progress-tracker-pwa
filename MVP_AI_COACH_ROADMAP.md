# NextStride AI Coach Roadmap (MVP-First)

## End Goal (Saved)
Build NextStride into a 24x7 AI assistant + coach that can guide users conversationally (eventually voice-first), reduce manual data entry, and help users consistently achieve meaningful life goals.

Design intent:
- Duolingo-like consistency and motivation loops
- Calm, non-animated experience
- High clarity, low friction, daily return behavior

## MVP Definition (What to build first)
MVP should prove one thing: users come back daily because the coach is useful.

MVP outcomes:
- Daily check-in in under 60 seconds
- Coach proposes the next 1-3 priority actions
- User can log progress quickly with guided prompts
- Weekly summary + next-week plan generated by coach

MVP non-goals:
- No avatar UI yet
- No complex real-time voice stack yet
- No heavy social or gamification system beyond simple streak + progress loops

## Product Flow (MVP)
1. Open app
2. Coach asks: "How are you today? What did you complete?"
3. User responds in natural language
4. Coach converts response into structured updates (goals, logs, action items)
5. Coach suggests next best action
6. End-of-day: quick reflection prompt

## Recommended Tech Direction

### Frontend
- Keep current React + Vite PWA for speed
- Add a Coach panel as primary interaction surface
- Keep existing forms as fallback editing tools

### AI Layer
- Add backend API endpoint that calls an LLM provider (do not call model directly from browser)
- Use structured JSON responses from the model for safe UI updates
- Create provider abstraction so model/vendor can be swapped later

### Data + Auth
- Move from local-only toward cloud-backed data
- Recommended: Supabase (Postgres + Auth + Row Level Security + edge functions)
- Keep import/export as backup even after cloud sync

### Hosting
- Recommended: Vercel or Netlify for frontend + serverless endpoints
- Keep static deployment path available for low-cost public demos

### Notifications
- MVP+1: web push reminders for daily check-ins and unfinished priorities

### Voice (Later)
- Phase after MVP: add speech-to-text + text-to-speech for coach mode
- Start with push-to-talk UX; real-time voice can come later

## Architecture Guardrails (Important)
- Keep domain logic separate from UI (goal engine, streak engine, coach engine)
- Version data schema to allow safe migrations
- Every AI action should be auditable (show "what the coach changed")
- Keep manual override everywhere (user always in control)

## Milestone Plan

### Milestone 1: Coach MVP (2-3 weeks)
- Add Coach chat panel
- Add natural-language to structured update pipeline
- Add daily check-in flow + next-action recommendation
- Add weekly summary generation

### Milestone 2: Cloud Sync (1-2 weeks)
- Auth (email/social)
- User data sync + conflict handling
- Multi-device continuity

### Milestone 3: Coach Reliability (1-2 weeks)
- Prompt/version management
- Evaluation prompts + guardrails
- Error handling and fallback to manual forms

### Milestone 4: Voice Assistant (later)
- STT/TTS
- Hands-free daily check-in
- Optional assistant persona tuning

## Success Metrics for MVP
- D7 retention improvement
- Daily check-in completion rate
- Average session duration
- % of users completing suggested next action
- Weekly active users returning 3+ days
