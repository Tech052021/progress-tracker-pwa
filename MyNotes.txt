Here is what the app can do right now, and what the GUI supports.

**Core App Capabilities (Current)**
1. Track progress across multiple areas using categories (for example Health, Career, Hobby).
2. Define measurable goals per category with:
- target value
- unit
- frequency period (day/week/month/year/hour/minute/target)
- optional victory date
3. Log day-to-day activity and see progress against goals.
4. View dashboard summary cards for key goals (for example cardio, protein, water, sleep).
5. Use a planner assistant to generate a draft category plan from natural-language input.
6. Regenerate the last planner draft safely before applying.
7. Keep planner-generated milestones and action steps in roadmap-style views.
8. Apply settings and goal updates with migration-safe behavior.
9. Run as a PWA-style web app (installable/offline-friendly shell behavior).

**Planner/Generation Capabilities**
1. Intake form supports goal type (general, weight/health, study/class, career/business).
2. Timeline uses target date input and computes internal timeline estimates.
3. Weight/health planning includes concrete targets (not vague):
- protein daily grams
- water daily liters
- sleep daily hours
4. Feasibility feedback gives a “gentle reality check” when targets are too aggressive.
5. Draft preview shows:
- suggested goals
- suggested milestones
- first action steps
6. Draft application supports append and replace behavior without destructive overwrites.

**GUI Capabilities**
1. Clean card-based dashboard with key metrics and streak context.
2. Settings modal with focused sections for long-term profile and categories/goals.
3. Category editor with inline goal rows and quick shortcuts:
- unit quick chips
- period quick chips
- inline hint text for health-style goals
4. Layered modal system:
- active popup owns scroll
- nested popup sizes are dynamically computed from parent modal size
- background layers stay locked when deeper popup is open
5. Responsive behavior for desktop and mobile viewport ranges.
6. Sticky/fixed action areas in forms where needed (Cancel/Apply, Generate flows).

**What It Does Not Fully Do Yet (Important)**
1. Full cloud account sync and cross-device auth flow are not complete yet.
2. Notification channels are not fully implemented.
3. “Goal mode” flexibility you proposed (none/single/multi long-term intent) is not yet implemented as a formal model.
